Project Summary

CuriousBooks is an online bookstore platform that enables customers to browse, search, and purchase books. The system provides a seamless shopping experience with features including user authentication, product search, shopping cart management, and secure checkout.

Corresponding Functional Requirements (Implementation)

FR ID Implements Implementation Description
FR-AUTH-001 NFR-AUTH-001 The LoginPage component provides a registration form. Form submission calls authApi.register() which sends a POST request to the Flask backend at /api/auth/register. The backend creates a new user record in the MySQL database and returns JWT tokens.
FR-AUTH-002 NFR-AUTH-002 Password validation is enforced on both frontend (HTML5 validation) and backend. The Flask auth.py route validates password requirements before hashing with bcrypt and storing in the users table.
FR-AUTH-003 NFR-AUTH-003 The LoginPage component calls authApi.login(email, password) which sends a POST to /api/auth/login. The Flask backend validates credentials against the MySQL database and returns JWT access/refresh tokens stored in localStorage.
FR-AUTH-004 NFR-AUTH-004 Logout calls authApi.logout() which clears JWT tokens from localStorage via the clearTokens() function in the API service, invalidating the user session.

FR-PROD-003 NFR-PROD-003 The BookSynopsisPage calls reviewsApi.getByBook(bookId) which sends a GET request to the Flask /api/reviews/book/:id endpoint. The backend joins the reviews and users tables to return reviewer names, ratings, dates, and text.

System Quality Attributes

Quality attributes and system constraints that define how the system should perform.

QA-PERF-001
Page Load Performance
The system shall load all pages within acceptable time limits to ensure a smooth user experience.
Target: Pages must load within 3 seconds on standard broadband connection
QA-PERF-002
Search Response Time
Search queries shall return results quickly to maintain user engagement and satisfaction.
Target: Search results displayed within 2 seconds
QA-SEC-001
Password Security
User passwords must be securely stored using industry-standard encryption methods.
Standard: Passwords hashed using bcrypt with minimum 10 salt rounds
QA-SEC-002
Session Management
User sessions shall timeout after period of inactivity to protect user accounts.
Target: Sessions expire after 30 minutes of inactivity
QA-USE-001
Responsive Design
The application shall be fully functional and visually appealing across all device sizes.
Support: Desktop, tablet, and mobile viewports (320px - 2560px)
QA-USE-002
Accessibility
The application shall meet accessibility standards to ensure usability for all users.
Standard: WCAG 2.1 Level AA compliance
QA-REL-001
System Availability
The system shall maintain high availability to ensure customers can access the store.
Target: 99.5% uptime during business hours
QA-SCL-001
Concurrent Users
The system shall support multiple simultaneous users without performance degradation.
Target: Support 100 concurrent users minimum
